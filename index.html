5 ways to think like a Programmer

1 Procedural Thinking - Taking Big Problems and breaking them into Smaller Pieces.
2 Abstract Thinking- Taking Simular Problems and solving them through Associated Reasoning.
3 Systematic Debugging - Systematically Identifing problems/potential problems, then tracking down thier Potential Causes until the problem has been fixed.
4 Technological Empathy - Understanding how the Computer/Code interprets things.
5 Systems Thinking - Taking a Large Idea, and breaking them into its Component Pieces.

By utilizing these 5 core mind sets, Programmers will take a complicated scope of work and breaking it into individual component pieces. As the Document is being coded, Large Scale problems are broken into smaller more manageable pieces to be solved, problemsthat have similar traits can be resolved via Abstract thinking, and the overall understanding of how Computers must be given the correct information/code in order to obtain the needed results are reviewed in order to reach a near final product. Once the near final product has been created, it must be tested, and debugged. By debugging this code, both real and potential problems can be identified and corrected, thus allowing a final product to be completed.

What is HTML

HTML stands for Hyper Text Markup Language, and is the main document type of the web, making it the true Heart of the WWW. The web itself is just a collection of documents that users can access, HTML allows those users to access all of these documents (Videos, Web sites, Social Media, ect.) HTML’s basic structure is made up of 3 core things.

HTML Basic Structure

1 Text Content-What you see
2 Markup-What the Document looks like
3 References to the other Documents- E.G. Imagines and Videos

HTML can be broken into 9 basic/linear pieces/steps, and a good video example of of this structure can be found here here.

1Document Tag - Identifies what type of HTML type this document is (Current standard is HTML 5).
2Opening HTML Tag - This works directly with the 9th piece/step, the Closing HTML tag. These tags identify and surround the rest of the entire Document.
3Opening Head Tag - This works directly with the 5th piece/step, and contains the meta-data, including the Java script, and CSS information.
4Title Tag - This is the Title of the page that will appear on the top of the web browser, and/or on the browser tab if multiple windows have been opened.
5Closing Head Tag - Please see piece/step 3.
6 Opening Body Tag - This works directly with piece 8 by identifying the start and end of the Content
7 Content Tag - This is the Heart and soul of the HTML document, and contains all of the coding that the end user’s wishes to access. The other 8 pieces/steps and important and required for the HTML document to work correctly, but are simple and do not change very often.
